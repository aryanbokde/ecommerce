How to use

Each module page is a self-contained checklist.

  • Columns: ID · Title · Precondition · Steps · Test Data · Expected Result · Type · Priority · Status.
  • Type: Positive happy path · Negative invalid input · Boundary limits/edge.
  • Click the Status pill to cycle Untested → Pass → Fail → Blocked. Saved in your browser (localStorage), per module.
  • Use the search box + Type/Priority filters at the top of each module to narrow the list.
